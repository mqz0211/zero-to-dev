#  Data Scientist Roadmap

**For people who want to:** turn raw data into insight and decisions —
less "build the model," more "figure out what question the data can
actually answer."
**Estimated time (part-time):** 10–14 months to internship-ready.

## Stage 0 — Fundamentals (2–3 weeks)
- [ ] [Computer basics](../FUNDAMENTALS/computer-basics.md)
- [ ] [Linux / command line basics](../FUNDAMENTALS/linux.md)
- [ ] [Problem-solving mindset](../FUNDAMENTALS/problem-solving.md)

## Stage 1 — Python & SQL for data (6 weeks)
- [ ] Python fundamentals (see [Software Engineer Stage 1](software-engineer.md#stage-1--pick-a-first-language-python-4-6-weeks))
- [ ] Pandas — the tool you'll use daily
- [ ] SQL: `SELECT`, `JOIN`, `GROUP BY`, subqueries — most company data
      lives in a database, not a CSV
- [ ] Jupyter notebooks

## Stage 2 — Statistics that actually get used (4–6 weeks)
- [ ] Descriptive statistics (mean, median, variance, distributions)
- [ ] Hypothesis testing and p-values (and their common misuse)
- [ ] Correlation vs. causation
- [ ] A/B testing fundamentals
- [ ] Sampling and bias

## Stage 3 — Data visualization & storytelling (2–3 weeks)
A model nobody understands doesn't get used. This stage is
underrated and directly job-relevant.
- [ ] Matplotlib/Seaborn/Plotly
- [ ] Choosing the right chart for the question
- [ ] Building a dashboard (Tableau or a free alternative)
- [ ] Writing a data story: what did you find, why does it matter, what
      should someone do about it

## Stage 4 — Machine learning fundamentals (6–8 weeks)
- [ ] Regression and classification (see [AI Engineer Stage 3](ai-engineer.md#stage-3--classical-machine-learning-4-6-weeks))
- [ ] Feature engineering
- [ ] Model evaluation metrics (accuracy isn't always the right one —
      learn precision/recall/F1 and when each matters)
- [ ] scikit-learn

## Stage 5 — Working with real, messy data (4 weeks)
This is what separates classroom data science from the job:
- [ ] Missing data, duplicates, inconsistent formatting
- [ ] Outlier detection
- [ ] Combining data from multiple sources
- [ ] Documenting your cleaning decisions (reproducibility matters)

## Stage 6 — A domain specialization (ongoing)
Data science applied to a specific field is more hireable than generic
data science:
- [ ] Business/product analytics
- [ ] Finance
- [ ] Healthcare
- [ ] Sports analytics
- [ ] Pick something you're genuinely curious about — it shows in your projects

## Stage 7 — Build a portfolio (ongoing)
- [ ] 2–3 projects using **real, messy, public datasets** (not the
      classic pre-cleaned Titanic/Iris datasets)
- [ ] Each project: a real question, a clear methodology, an honest
      discussion of limitations
- [ ] At least one project with a dashboard or visualization someone
      non-technical could understand
- [ ] See [`CAREERS/portfolio.md`](../CAREERS/portfolio.md)

##  You're ready for an internship interview when you can:
- Explain a p-value correctly to someone who's never heard of one
- Defend a modeling choice you made, including its trade-offs
- Turn a vague business question into a specific, answerable data question
