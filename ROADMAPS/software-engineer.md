#  Software Engineer Roadmap

**For people who want to:** build applications, websites, tools, and
systems for a living.
**Estimated time (part-time, alongside school):** 8–12 months to
internship-ready.

> New here? Read [`FUNDAMENTALS/`](../FUNDAMENTALS/) first if terms like
> "RAM," "terminal," or "API" aren't familiar yet.

## Stage 0 — Fundamentals (2–3 weeks)
- [ ] [Computer basics](../FUNDAMENTALS/computer-basics.md)
- [ ] [Linux / command line basics](../FUNDAMENTALS/linux.md)
- [ ] [How the internet works](../FUNDAMENTALS/networking.md)
- [ ] [Problem-solving mindset](../FUNDAMENTALS/problem-solving.md)
- [ ] Set up: a code editor (VS Code), Git, and GitHub account

## Stage 1 — Pick a first language: Python (4–6 weeks)
Python first, because its syntax gets out of the way of learning
*how to think*, not just how to code.
- [ ] Variables, data types, operators
- [ ] Conditionals (`if`/`else`) and loops (`for`/`while`)
- [ ] Functions and scope
- [ ] Lists, dictionaries, and basic data structures
- [ ] Reading/writing files
- [ ] Error handling (`try`/`except`)

**Build as you go:** see [`PROJECTS/beginner-projects.md`](../PROJECTS/beginner-projects.md)
for the Python project ladder (Calculator → Password Manager → Web
Scraper → AI Assistant).

## Stage 2 — Computer Science fundamentals (4–6 weeks)
This is what separates "can copy-paste code" from "can actually solve
new problems" — and it's what technical interviews test.
- [ ] Big O notation (why efficiency matters)
- [ ] Arrays, linked lists, stacks, queues
- [ ] Hash maps / dictionaries (how they actually work underneath)
- [ ] Trees and graphs (basic traversal: BFS/DFS)
- [ ] Sorting algorithms (bubble, merge, quick — and *why* each exists)
- [ ] Recursion

**Practice:** [NeetCode](https://neetcode.io) (free tier) and
[LeetCode](https://leetcode.com) (free tier), 2–3 problems a week,
easy → medium.

## Stage 3 — Git & GitHub properly (1 week)
Not just `git add . && git commit` — actually understanding it.
- [ ] Commits, branches, merging
- [ ] Resolving merge conflicts
- [ ] Pull requests and code review
- [ ] Writing a good commit message and README
- [ ] See [`CAREERS/github-guide.md`](../CAREERS/github-guide.md) for
      making your profile look professional

## Stage 4 — Web fundamentals (4–6 weeks)
Even if you don't become a web developer, every software engineer runs
into HTTP, APIs, and databases constantly.
- [ ] HTML/CSS basics
- [ ] JavaScript basics
- [ ] What an API is, and how to call one (`fetch`/`requests`)
- [ ] What a database is; basic SQL (`SELECT`, `WHERE`, `JOIN`)
- [ ] Build a small full-stack project — see [`PROJECTS/intermediate-projects.md`](../PROJECTS/intermediate-projects.md)

## Stage 5 — A second language + specialization (6–8 weeks)
Pick based on interest — this is where paths start to diverge:
- [ ] **Backend-leaning:** learn Java, Go, or C# — statically typed,
      widely used in industry
- [ ] **Systems-leaning:** learn C or C++ — understand memory management
- [ ] **Product-leaning:** go deeper into JavaScript/TypeScript + a
      framework (React)

## Stage 6 — Build a portfolio-worthy project (4–6 weeks)
One real project beats five tutorial clones. It should:
- [ ] Solve a real (even small) problem
- [ ] Have a clean, well-documented GitHub repo (see [`CAREERS/portfolio.md`](../CAREERS/portfolio.md))
- [ ] Include tests
- [ ] Be deployed somewhere live if it's a web app

See [`PROJECTS/advanced-projects.md`](../PROJECTS/advanced-projects.md)
for ideas sequenced by difficulty.

## Stage 7 — Internship-ready (ongoing)
- [ ] Resume built around your projects (see [`CAREERS/portfolio.md`](../CAREERS/portfolio.md))
- [ ] 2–3 solid projects on GitHub, pinned, with good READMEs
- [ ] Comfortable explaining any line of your own code out loud
- [ ] Practicing interview-style problems weekly
- [ ] See [`CAREERS/internships.md`](../CAREERS/internships.md) for
      where and how to apply

##  You're ready for an internship interview when you can:
- Explain what happens, step by step, when you type a URL and hit enter
- Write a function that reverses a linked list without looking it up
- Walk someone through a project on your GitHub, including a decision
  you'd make differently now
