#  AI Engineer Roadmap

**For people who want to:** build, fine-tune, and deploy machine
learning and AI systems.
**Estimated time (part-time):** 10–14 months to internship-ready.

> This path assumes comfort with high-school-level algebra. If math
> feels rusty, that's normal — the resources below rebuild it as you go.

## Stage 0 — Fundamentals (2–3 weeks)
- [ ] [Computer basics](../FUNDAMENTALS/computer-basics.md)
- [ ] [Linux / command line basics](../FUNDAMENTALS/linux.md)
- [ ] [Problem-solving mindset](../FUNDAMENTALS/problem-solving.md)

## Stage 1 — Python for data (4–6 weeks)
- [ ] Python fundamentals (see [Software Engineer Stage 1](software-engineer.md#stage-1--pick-a-first-language-python-4-6-weeks))
- [ ] NumPy (arrays, vectorized operations)
- [ ] Pandas (loading, cleaning, exploring data)
- [ ] Matplotlib/Seaborn (visualizing data)

## Stage 2 — The math you actually need (6–8 weeks)
You don't need a math degree — you need working intuition for:
- [ ] Linear algebra: vectors, matrices, matrix multiplication
- [ ] Calculus: derivatives, gradients (what "gradient descent" means)
- [ ] Probability & statistics: distributions, mean/variance, Bayes' theorem
- [ ] **Learn it visually first** — see [`RESOURCES/youtube.md`](../RESOURCES/youtube.md)
      for the best visual math explainers before touching a textbook

## Stage 3 — Classical machine learning (4–6 weeks)
- [ ] What "training" a model actually means
- [ ] Regression vs. classification
- [ ] Decision trees, random forests
- [ ] Overfitting/underfitting, train/test/validation splits
- [ ] scikit-learn — build your first real models

**Build as you go:** small end-to-end projects (spam classifier, price
predictor) — see [`PROJECTS/intermediate-projects.md`](../PROJECTS/intermediate-projects.md)

## Stage 4 — Neural networks & deep learning (6–8 weeks)
- [ ] What a neuron/layer actually computes
- [ ] Backpropagation (conceptually, then in code)
- [ ] Activation functions, loss functions, optimizers
- [ ] PyTorch or TensorFlow basics
- [ ] Convolutional networks (CNNs) for images

## Stage 5 — Modern AI: transformers & LLMs (6–8 weeks)
This is the fastest-moving part of the field — expect to keep learning
past this roadmap.
- [ ] Tokens, embeddings
- [ ] Attention mechanism and transformer architecture
- [ ] How training vs. fine-tuning vs. prompting differ
- [ ] Using LLM APIs in a real project (prompt design, function calling)
- [ ] Fine-tuning a small open model

**Build:** understand a transformer by building a minimal one from
scratch — a genuinely excellent, resume-worthy project. See
[`PROJECTS/advanced-projects.md`](../PROJECTS/advanced-projects.md).

## Stage 6 — MLOps basics (3–4 weeks)
Building a model is different from running one in production.
- [ ] Model versioning and experiment tracking
- [ ] Serving a model behind an API
- [ ] Basic monitoring (is the model still accurate over time?)

## Stage 7 — Build a portfolio (ongoing)
- [ ] 2–3 end-to-end projects: data → model → evaluation → (ideally) a
      deployed demo
- [ ] At least one project that goes beyond "followed a tutorial" — a
      dataset or problem you chose yourself
- [ ] Clear README explaining your approach and results, including what
      didn't work
- [ ] See [`CAREERS/portfolio.md`](../CAREERS/portfolio.md)

##  You're ready for an internship interview when you can:
- Explain overfitting to someone with no ML background, using an analogy
- Walk through your own project's model choices and why you made them
- Read someone else's model training code and explain what it's doing
