#  Web Developer Roadmap

**For people who want to:** build websites and web applications — the
fastest path from "learning to code" to "shipping something real people
can visit."
**Estimated time (part-time):** 6–9 months to internship-ready.

## Stage 0 — Fundamentals (2–3 weeks)
- [ ] [Computer basics](../FUNDAMENTALS/computer-basics.md)
- [ ] [Linux / command line basics](../FUNDAMENTALS/linux.md)
- [ ] [How the internet works](../FUNDAMENTALS/networking.md) — essential
      for this path specifically
- [ ] [Problem-solving mindset](../FUNDAMENTALS/problem-solving.md)

## Stage 1 — HTML & CSS (3–4 weeks)
- [ ] HTML structure and semantic tags
- [ ] CSS selectors, the box model, Flexbox, Grid
- [ ] Responsive design (mobile vs. desktop layouts)
- [ ] Build 3–4 static pages from scratch (no frameworks yet)

## Stage 2 — JavaScript (6–8 weeks)
- [ ] Variables, functions, control flow (same fundamentals as any language)
- [ ] The DOM — how JavaScript changes what's on the page
- [ ] Events (clicks, form submissions)
- [ ] Async JavaScript: promises, `async`/`await`, `fetch`
- [ ] Array methods (`map`, `filter`, `reduce`) — you will use these constantly

**Build as you go:** see [`PROJECTS/beginner-projects.md`](../PROJECTS/beginner-projects.md)
for a sequenced front-end project ladder.

## Stage 3 — Git, GitHub, and deployment (1–2 weeks)
- [ ] Git basics (see [Software Engineer Stage 3](software-engineer.md#stage-3--git--github-properly-1-week))
- [ ] Deploy a static site (GitHub Pages, Netlify, or Vercel — all free)
- [ ] Custom domains (optional, but satisfying)

## Stage 4 — A front-end framework (6–8 weeks)
- [ ] Pick React (most jobs, most learning resources) — Vue and
      Svelte are excellent too, but React has the largest hiring surface
- [ ] Components, props, state
- [ ] Hooks (`useState`, `useEffect`)
- [ ] Routing between pages
- [ ] Fetching and displaying data from an API

## Stage 5 — Backend basics (6–8 weeks)
Front-end-only developers are less hireable than full-stack-capable ones.
- [ ] Node.js + Express (JavaScript backend — reuses what you already know)
- [ ] REST API design
- [ ] Databases: start with SQLite/PostgreSQL, learn basic SQL
- [ ] Authentication basics (how login actually works — sessions vs. tokens)

## Stage 6 — Build a full-stack project (4–6 weeks)
- [ ] A project with a real front end, real backend, and real database
- [ ] Deployed and actually usable at a live URL
- [ ] See [`PROJECTS/intermediate-projects.md`](../PROJECTS/intermediate-projects.md)
      and [`PROJECTS/advanced-projects.md`](../PROJECTS/advanced-projects.md)

## Stage 7 — Build a portfolio (ongoing)
- [ ] A personal portfolio site (this is your first real web dev project
      and also your resume)
- [ ] 2–3 deployed, working projects with clean GitHub repos
- [ ] See [`CAREERS/portfolio.md`](../CAREERS/portfolio.md)

## You're ready for an internship interview when you can:
- Explain what happens between clicking a link and a new page rendering
- Build a small feature live, thinking out loud, without panicking
- Talk through a deployed project on your portfolio site, including a
  bug you hit and how you debugged it
