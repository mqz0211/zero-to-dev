# ☁️ Cloud Engineer Roadmap

**For people who want to:** deploy, run, and scale infrastructure and
applications on cloud platforms.
**Estimated time (part-time):** 8–12 months to internship-ready.

## Stage 0 — Fundamentals (2–3 weeks)
- [ ] [Computer basics](../FUNDAMENTALS/computer-basics.md)
- [ ] [Linux / command line basics](../FUNDAMENTALS/linux.md) — non-negotiable
      for this path
- [ ] [How the internet works](../FUNDAMENTALS/networking.md) — also
      non-negotiable
- [ ] [Problem-solving mindset](../FUNDAMENTALS/problem-solving.md)

## Stage 1 — Programming basics (4–6 weeks)
You don't need to be a software engineer, but you do need to script:
- [ ] Python fundamentals (see [Software Engineer Stage 1](software-engineer.md#stage-1--pick-a-first-language-python-4-6-weeks))
- [ ] Bash scripting — automating repetitive ops tasks

## Stage 2 — Networking, deeper (4 weeks)
- [ ] IP addressing and subnetting
- [ ] DNS in depth
- [ ] Load balancers, reverse proxies
- [ ] HTTPS/TLS basics — how encryption in transit actually works

## Stage 3 — Linux administration (4–6 weeks)
- [ ] Users, permissions, and services (systemd)
- [ ] Package management
- [ ] Monitoring basic system resources (CPU, memory, disk)
- [ ] SSH — key-based authentication, not just passwords

## Stage 4 — Git & version control for infrastructure (1–2 weeks)
- [ ] Git basics (see [Software Engineer Stage 3](software-engineer.md#stage-3--git--github-properly-1-week))
- [ ] Why "infrastructure as code" matters (your infra config lives in
      Git, just like application code)

## Stage 5 — Containers (4 weeks)
- [ ] Docker: images, containers, Dockerfiles
- [ ] Docker Compose for multi-container setups
- [ ] Why containers solve "it works on my machine"

## Stage 6 — A cloud platform (6–8 weeks)
Pick one to go deep on first (concepts transfer between them later):
- [ ] AWS, Azure, or Google Cloud — AWS has the largest market share
      and free-tier learning resources
- [ ] Core services: compute (VMs), storage, networking (VPC), and a
      managed database
- [ ] IAM — identity and access management (who can do what, and why
      this is the #1 source of real-world security incidents)

## Stage 7 — CI/CD and automation (4 weeks)
- [ ] What a CI/CD pipeline actually does
- [ ] GitHub Actions — build, test, and deploy automatically on push
- [ ] Basic Infrastructure-as-Code (Terraform) — define infrastructure
      in a file instead of clicking through a console

## Stage 8 — Kubernetes basics (4–6 weeks, optional but valuable)
- [ ] What container orchestration solves that Docker alone doesn't
- [ ] Pods, deployments, services
- [ ] This is a deep topic — basic fluency is enough to start

## Stage 9 — Build a portfolio (ongoing)
- [ ] Deploy a real application to the cloud, end to end, with CI/CD
- [ ] Document your architecture with a diagram (even a simple one)
- [ ] A GitHub repo showing your Infrastructure-as-Code, not just
      manual click-through setup
- [ ] See [`CAREERS/portfolio.md`](../CAREERS/portfolio.md)

## ✅ You're ready for an internship interview when you can:
- Explain the difference between a VM and a container
- Walk through what happens when your CI/CD pipeline runs, step by step
- Explain why least-privilege IAM matters, with a concrete example
